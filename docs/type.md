# 커밋 시 앞에 붙여서 변경사항 타입 명시

## 타입: 커밋 메시지 형식으로 커밋

| 타입       | 설명                                                  |
| ---------- | ----------------------------------------------------- |
| `feat`     | 새로운 기능 추가                                      |
| `fix`      | 버그 수정                                             |
| `docs`     | 문서 수정 (README, 주석 등)                           |
| `style`    | 코드 포맷 변경 (기능 변화 없음, 들여쓰기/세미콜론 등) |
| `refactor` | 코드 리팩토링 (기능 변화 없음)                        |
| `test`     | 테스트 코드 추가/수정                                 |
| `chore`    | 빌드 업무, 패키지 업데이트 등 기타 작업               |
| `perf`     | 성능 개선                                             |
| `ci`       | CI/CD 설정 관련 변경                                  |
| `build`    | 빌드 시스템 또는 외부 의존성 변경 (예: webpack, npm)  |
